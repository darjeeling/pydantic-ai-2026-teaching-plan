## 결정된 기준

- 말투: 혼합. 설명문은 자연스러운 문어체("~다"), 따옴표 안 강사 발화는 진짜 구어체("~요/~죠").
- 범위: 전체 (lessons 6 + slides 6 + exercises 6 + README).
- 깊이: 말투 + 흐름 재배열 + 회차 간 중복 통합.

## 사실관계 (이미 양호, 손대지 않음)

- 모델명 일관적: `openai:gpt-5.2`, `openrouter:anthropic/claude-sonnet-4.6`, `google:gemini-3-pro-preview`, `bedrock:anthropic.claude-sonnet-4-5-...`.
- lessons가 참조하는 examples 코드/함수/클래스 전부 실재.
- lessons ↔ slides ↔ exercises ↔ README 회차 매핑 정확.

## 스타일 가이드 (전 파일 공통)

1. 설명문: 문어체 유지하되 번역투 제거. 금지 패턴: "~를 이해", "~점을 설명한다", "~로 다룬다", "~을 만든다"의 기계적 반복, 매 문장 동일 문말.
2. 강사 발화(따옴표): 합쇼체 격식 → 가벼운 구어체. 예) "~만들 수 있습니다" → "~만들 수 있어요".
3. 섹션 라벨 통일: "강사가 말할 포인트" / "강사가 강조할 점" / "강사가 강조할 문장" / "말할 내용" 혼용 → 용도별 2종으로 통일.
   - 개념 짚기: "수업에서는 이렇게 짚어 준다."
   - 그대로 말할 대사: "이렇게 말한다." + 따옴표 구어체.
4. 문장 시작/끝의 패턴 반복 깨기. 한 단락에 같은 구문 3회 이상 금지.

## 흐름 재배열

- 02/03/04회차: 현재 "사례 → 120분 진행안 → 도입 이야기" 역순. → "도입 이야기 → 사례 → 120분 진행안 → 개념"으로.
- 03회차: 위험 코드 예시(10-25줄) → "따라서 핵심은"(26줄) 사이 논리 전개 보강.
- 04회차: Evals 개념 순서(Dataset/Case/Evaluator)와 라이브코딩 코드 읽기 순서(task→dataset→evaluator) 불일치 → 한쪽으로 맞춤.

## 회차 간 중복 통합

- LangChain/LangGraph 비교: 1회차에 본론, 4회차는 한 줄 참조로 축소.
- Human-in-the-loop: 4회차=reviewer agent, 5회차=durable 승인 대기/재개로 역할 분리. 4회차의 5회차 미리보기 축소.
- WSL 주의: 0회차 본론, 3회차는 한 줄 참조.
- retry vs durable 표: 5회차 내 중복 표 한 번으로 통합.
- 5회차 HITL 설계: 산문 설명 + 동일 내용 pseudocode 중복 → 하나로.
